# Feature Roadmap

## Phase 1: Mobile Prototype

Build the app as a simple phone-friendly web app.

Features:

- Home screen
- Bottom navigation
- Customer list
- Job list
- Add job form
- Job detail screen
- Invoice view
- Local browser storage

## Phase 2: Better Job Management

Improve the work tracking flow.

Features:

- Job statuses
- Paid and unpaid filters
- Calendar-style job list
- Customer history
- Search customers and jobs
- Notes for each job

## Phase 3: Customer Booking

Add a customer-facing booking flow.

Features:

- Service selection
- Customer contact form
- Job location form
- Requested date and time
- Job notes
- Confirmation screen

## Phase 4: Payments and Messages

Add business tools after the basic app works.

Features:

- Deposit payment integration
- Invoice payment link
- Text confirmations
- Email confirmations
- Reminder messages

## Phase 5: Backend and Accounts

Move from local storage to a real backend.

Possible options:

- Node.js and Express
- SQLite for simple storage
- PostgreSQL for larger use
- Supabase or Firebase if faster setup is needed

## Phase 6: Installable App

Turn the web app into an installable phone app.

Features:

- PWA manifest
- App icon
- Offline shell
- Save to home screen
- Optional app store wrapper later
